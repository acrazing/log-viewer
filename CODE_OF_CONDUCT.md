# Code of Conduct

RawLens should be a useful, low-friction project for developers who are debugging messy data. Everyone
participating in issues, pull requests, discussions, and reviews is expected to help keep that space
clear and respectful.

## Expected Behavior

- Be kind, direct, and constructive.
- Assume good intent, especially when discussing bugs or design tradeoffs.
- Keep feedback about code, behavior, docs, or product decisions rather than personal traits.
- Respect different experience levels and communication styles.
- Use redacted or synthetic data when discussing logs, payloads, or security issues.

## Unacceptable Behavior

- Harassment, threats, insults, or discriminatory language.
- Publishing private information, secrets, tokens, logs, or customer data.
- Repeated off-topic comments after maintainers ask to refocus.
- Security disclosures in public issues before maintainers have a chance to triage privately.

## Enforcement

Maintainers may edit, hide, or delete comments; close issues or pull requests; or block participants who
make the project unsafe or unproductive.

If you need to report conduct concerns, open a minimal issue asking for a private maintainer contact. Do
not include sensitive personal details in a public issue.
